# 供应商价格本管理方案

## 1. 目标

将供应商价格本从“离线 Excel 参考表”升级为系统内正式可维护、可导入、可追溯的采购参考能力，用于支撑：

- 物料默认供应商与备选供应商管理
- 供应商报价对比
- 最近采购价格参考
- 高风险供应物料治理
- 后续采购任务与补货建议

适用数据来源：

- [供应商价格本_20260323170838.xlsx](<user-provided-local-file>)

## 2. 当前现状

当前系统已具备：

- 物料供应商基础关系
- 默认供应商
- 来源平台
- 最近采购参考
- 物料详情中的供应商卡片

当前系统缺少：

- 正式的供应商报价台账
- 同一物料多供应商价格比较
- 折扣与最近成交价结构化管理
- 供应商价格本专用导入
- 采购侧对报价信息的直接使用

结论：

- 当前系统“能承接”这类数据
- 但还没有把它变成正式功能

## 3. 源文件结构判断

该价格本实际字段包括：

- 往来单位编号
- 往来单位名称
- 商品编号
- 商品名称
- 单位
- 规格
- 型号
- 指定折前单价
- 指定折扣
- 最近采购折前价格
- 最近采购折扣
- 最近采购时间

业务含义判断：

- `指定折前单价 + 指定折扣`：更接近当前报价或协议价
- `最近采购折前价格 + 最近采购折扣 + 最近采购时间`：更接近历史实际采购参考

因此，这份表不应只当作“供应商通讯录”，而应视为：

- 供应商报价主档
- 采购参考价来源

## 4. 建议功能定位

建议将该能力命名为：

- `供应商价格本`
- 或 `供应商报价管理`

并定位为：

- 物料主数据下的采购参考子模块
- 不是财务正式结算模块
- 也不是采购订单替代品

其作用是：

- 告诉用户“这个料可以找谁买”
- 告诉用户“最近参考价格是多少”
- 告诉用户“当前谁是默认来源、谁是备选来源”

## 5. 数据模型方案

建议新增正式表：

- `material_supplier_prices`

推荐字段：

- `id`
- `material_id`
- `supplier_name`
- `supplier_code`
- `supplier_type`
- `source_platform`
- `shop_name`
- `shop_url`
- `purchase_url`
- `quoted_price`
- `quoted_discount`
- `effective_price`
- `last_purchase_price`
- `last_purchase_discount`
- `last_purchase_effective_price`
- `last_purchase_at`
- `unit`
- `spec`
- `model`
- `currency`
- `is_default`
- `is_active`
- `notes`
- `raw_source`
- `created_at`
- `updated_at`

说明：

- `effective_price = quoted_price * quoted_discount`
- `last_purchase_effective_price = last_purchase_price * last_purchase_discount`
- `raw_source` 可保留原始导入字段，便于回溯

## 6. 与现有数据的关系

当前系统已有：

- `materials.default_supplier_id`
- `material_suppliers`

建议关系如下：

- `material_suppliers`：偏供应商主关系与采购来源信息
- `material_supplier_prices`：偏报价与历史采购价格信息

可选策略：

1. 保守方案
- 保留 `material_suppliers`
- 新增 `material_supplier_prices`
- 两者并行

2. 收敛方案
- 后续逐步将 `material_suppliers` 升级为“关系主表”
- `material_supplier_prices` 作为价格子表

当前阶段建议采用：

- 保守方案

## 7. 核心功能范围

### 7.1 价格本导入

新增专用导入能力：

- `POST /api/material-supplier-prices/import/preview`
- `POST /api/material-supplier-prices/import/commit`

导入逻辑：

- 自动识别真实表头
- 优先按 `商品编号` 匹配物料
- 其次按 `商品名称 + 型号/规格` 辅助匹配
- 预校验显示：
  - 可新增
  - 可更新
  - 未匹配物料
  - 重复报价

### 7.2 物料详情展示

在物料详情页新增：

- 默认采购来源
- 报价列表
- 最近采购价
- 报价更新时间
- 平台/店铺/链接
- 是否唯一供应来源

### 7.3 报价比较

同一物料可展示多个供应商：

- 当前报价
- 最近采购价
- 折后价
- 最近采购时间
- 默认来源标记

### 7.4 与高风险供应预警联动

价格本进入高风险供应判断时，可增强：

- 唯一供应商判断
- 长时间无采购记录判断
- 无有效报价判断
- 默认供应来源失效判断

### 7.5 采购动作支持

后续可供采购动作直接使用：

- 建议找谁买
- 最近成交价参考
- 当前报价参考
- 店铺或采购链接跳转

## 8. 页面方案

### 8.1 物料详情页

新增区块：

- `供应商报价`

每条记录展示：

- 供应商名称
- 类型
- 平台
- 当前报价
- 折扣
- 折后价
- 最近采购价
- 最近采购时间
- 默认标记
- 链接入口

### 8.2 物料列表

后续可补列：

- 默认供应商
- 来源平台
- 最近采购价
- 报价更新时间

### 8.3 导入页面

建议新增专用入口：

- `物料 > 导入 > 供应商价格本`

模式与当前：

- 模板导入
- 库存状况表导入

保持一致。

### 8.4 治理页

后续可新增：

- `供应商价格治理`

用于查看：

- 未匹配物料
- 无默认来源
- 无有效报价
- 价格长期未更新

## 9. 导入映射规则

建议采用以下字段映射：

- `往来单位名称 -> supplier_name`
- `往来单位编号 -> supplier_code`
- `商品编号 -> material.code`
- `商品名称 -> material.name`
- `单位 -> unit`
- `规格 -> spec`
- `型号 -> model`
- `指定折前单价 -> quoted_price`
- `指定折扣 -> quoted_discount`
- `最近采购折前价格 -> last_purchase_price`
- `最近采购折扣 -> last_purchase_discount`
- `最近采购时间 -> last_purchase_at`

计算字段：

- `effective_price = quoted_price * quoted_discount`
- `last_purchase_effective_price = last_purchase_price * last_purchase_discount`

## 10. 导入策略建议

### 10.1 Preview / Commit 双阶段

必须采用：

- 先预览
- 再提交

不要直接写库。

### 10.2 匹配优先级

推荐顺序：

1. `商品编号`
2. `商品名称 + 型号`
3. `商品名称 + 规格`

若仍不唯一：

- 标记为待人工确认

### 10.3 默认来源设置

建议规则：

- 若当前物料无默认供应商，导入第一条可自动建议为默认
- 若已有默认供应商，不自动覆盖，只作为报价补充

## 11. 风险与边界

### 11.1 不是财务结算依据

该价格本应定位为：

- 采购参考价
- 供应商参考信息

不应直接作为：

- 财务应付结算唯一依据

### 11.2 价格可能不是即时有效

因为 Excel 里的“指定价格”和“最近采购价”都可能过期，所以系统应允许：

- 标记为过期
- 标记为待更新

### 11.3 同一物料多个报价是正常的

不能再按“一物料只能有一个供应商”设计。

## 12. 实施顺序

### 第一阶段

- 新增数据表
- 新增导入 preview/commit
- 物料详情页展示报价

### 第二阶段

- 默认供应来源与报价联动
- 高风险供应预警增强
- 供应商价格治理页

### 第三阶段

- 与采购任务联动
- 与补货建议联动
- 报价失效提醒

## 13. 验收标准

满足以下条件时，可认为第一阶段完成：

1. 能识别并导入该 Excel 价格本
2. 能按商品编号匹配现有物料
3. 能预览新增/更新/未匹配结果
4. 物料详情页可展示报价记录
5. 同一物料支持多供应商报价
6. 能看到默认来源与最近采购价
7. 不会覆盖现有真实业务库存和单据数据

## 14. 最终结论

这份供应商价格本里的内容有必要在系统里做正式管理，而且值得做成：

- `供应商报价管理`
- `采购参考价管理`

当前系统已经有承接基础，最合理的下一步不是继续靠 Excel 离线维护，而是：

- 先把这类表做成专用导入
- 再把报价信息挂到物料详情与高风险供应治理里

这样后续采购、补货和供应风险判断都会更有依据。
