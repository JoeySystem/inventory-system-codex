# 第一阶段验收清单

## 目标

第一阶段的目标不是重写业务单据，而是完成信息架构收口，让系统从旧的模块并列导航，过渡到以 `收 / 发 / 转 / 盘 / 生产 / 报表 / 系统` 为一级语言的可用入口结构。

## 验收范围

- 左侧导航结构
- 顶部标题与副标题
- 顶部全局搜索
- 仪表盘信息架构
- 七个一级工作台页
- 旧页面的归类说明与术语统一

## 验收项

### 1. 一级导航

- 左侧导航已调整为：
  - `仪表盘`
  - `物料`
  - `收`
  - `发`
  - `转`
  - `盘`
  - `生产`
  - `报表`
  - `系统`
- 导航点击后能进入对应工作台或页面。
- 权限不足时，对应入口不显示。

### 2. 顶部页面信息

- 每个页面顶部标题与副标题已按新架构显示。
- 旧页面顶部标题已替换为新口径：
  - `仓库资料与执行`
  - `销售发货`
  - `分析看板`
  - `正式报表`
  - `SOP工艺`
  - `BOM结构`
  - `生产工单`
  - `用户与权限`

### 3. 顶部搜索

- 顶部搜索框文案已改为“搜索主数据、动作入口或生产入口”。
- 输入关键词后会出现分流建议。
- 搜索结果至少支持：
  - 物料主档搜索
  - `收`
  - `发`
  - `转`
  - `盘`
  - `生产`
  - `报表`
  - `系统`
- 回车时优先执行第一条建议。

### 4. 仪表盘

- 仪表盘已按四层语言重组：
  - `风险层`
  - `动作层`
  - `核算层`
  - `管理层`
- 首页中已能明确区分：
  - 风险摘要
  - 主档与库存概览
  - 动作入口
  - 最近动态
  - 当日摘要
- 低库存区域已带数字说明。

### 5. 工作台页面

- `收 / 发 / 转 / 盘 / 生产 / 报表 / 系统` 七个工作台页均已具备统一结构：
  - 页面说明
  - 关键词标签
  - 常见场景
  - 后续标准单据
  - 分组入口卡片
  - 本阶段说明
- 工作台不再只是跳转页，而具备阶段性业务首页属性。

### 6. 旧页面归类说明

- 旧页面顶部均已出现“第一阶段归类说明”。
- 说明内容应能明确回答：
  - 当前页面在新架构中归属到哪里
  - 当前阶段为什么仍复用旧页
  - 后续将向哪个方向拆分

### 7. 术语一致性

- 面向用户可见的高频术语已统一为新口径。
- 重点检查：
  - `销售发货`
  - `生产工单`
  - `库存余额`
  - `动作流水`
  - `分析看板`
  - `正式报表`
  - `仓库资料与执行`
- 允许保留少量“旧页来源说明”类文案，例如：
  - 原发货管理
  - 原仓库执行页
  - 原报表中心

### 8. 报表与统计口径提示

- `分析看板` 页面已显式标注：
  - 动作层
  - 核算层
  - 管理层
- `正式报表` 页面已显式标注：
  - 核算层
  - 动作层
  - 经营层
- 空状态文案不再只显示“暂无数据”，而是说明当前是哪类数据为空。

## 非验收项

以下内容不属于第一阶段必须完成范围：

- 库存模型重构
- 单据模型重构
- 收发转盘标准单据真正拆库
- 仓库资料页与仓库执行页彻底分离
- 生产工单快照
- 进销存公式层面重算

这些内容应放到第二阶段及之后执行。

## 当前收尾建议

- 先按本清单做一次页面走查。
- 确认所有一级入口都能正常进入。
- 确认旧页面顶部说明都能解释清楚当前归属。
- 确认顶部搜索的分流建议符合预期。
- 确认首页四层语言已经能被实际用户理解。

## 通过标准

满足以下条件即可判定第一阶段通过：

- 一级导航全部切换到新架构语言
- 七个工作台入口全部可用
- 仪表盘、顶部搜索、旧页说明三处已统一到新口径
- 用户在不看文档的情况下，也能大致理解：
  - 去哪里收
  - 去哪里发
  - 去哪里转
  - 去哪里盘
  - 去哪里做生产
  - 去哪里看报表
  - 去哪里改系统配置
