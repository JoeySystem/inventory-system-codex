# 高风险供应物料预警可开发规则

## 1. 目的

在现有 `低库存预警` 与 `保供套数预警` 之外，新增一套 `高风险供应预警` 规则，用于提前识别：

- 唯一供应商物料
- 长交期物料
- 无替代料物料
- 关键件
- 会直接卡住 BOM / 工单 / 成品保供的风险物料

目标不是替代现有预警，而是补一层更贴近真实供应风险的治理能力。

---

## 2. 适用场景

重点适用于以下物料：

- 原厂直采且供应商唯一的器件
- 交期长于 7 天、14 天、30 天的关键料
- 采购渠道集中在单一平台或单一店铺的物料
- 无替代料、不可替代工艺料
- 一旦缺料就会卡住整机或关键半成品的物料

典型案例：

- PCB 板上的特定贴片负载
- 关键射频器件
- 特殊线缆连接器
- 定制模块
- 交期长且只有一家供货方的治具或组件

---

## 3. 设计原则

### 3.1 双预警继续保留

系统继续保留：

- 物料级预警：`安全库存 / 最低库存 / 补货点`
- 套数级预警：`目标保供套数`

新增：

- 供应风险预警：`高风险供应料`

### 3.2 高风险预警不是只看库存

高风险预警需要同时看：

- 当前库存
- 交期
- 供应商数量
- 是否关键件
- 是否有替代料
- 是否会卡住保供套数

### 3.3 先做规则型，后做智能型

第一阶段先落规则型判断，不依赖复杂预测模型。  
后续再考虑引入：

- 日均消耗
- 采购历史
- 实际交期波动
- 季节性风险

---

## 4. 数据模型扩展

### 4.1 物料主档新增字段

建议在 `materials` 增加以下字段：

- `is_single_source` INTEGER DEFAULT 0
  - 是否唯一供应商
- `coverage_days_target` REAL DEFAULT 0
  - 目标保供天数
- `supply_risk_level` TEXT DEFAULT 'normal'
  - 可选值：`normal / medium / high / critical`
- `supply_risk_notes` TEXT
  - 风险说明，例如“原厂直采，常规交期 10-14 天”

说明：

- `lead_time_days`、`is_key_part` 当前已有，可直接复用
- `default_supplier_id` 当前已有，可直接复用
- `material_substitutions` 当前已有，可用于判断是否有替代料

### 4.2 供应商关系复用字段

当前 `material_suppliers` 已有或已扩展：

- `supplier_type`
- `source_platform`
- `lead_time_days`
- `purchase_url`
- `contact_person`
- `contact_phone`
- `manufacturer_name`

这些字段可以直接参与供应风险计算，无需重复建模。

### 4.3 计算辅助字段

不建议先落物理表，第一阶段可按接口动态计算：

- `supplierCount`
- `hasDefaultSupplier`
- `hasSubstitution`
- `effectiveLeadTimeDays`
- `coverageDays`
- `coverageRiskGapDays`
- `riskScore`

---

## 5. 核心判定规则

## 5.1 唯一供应商判定

判定优先级：

1. 若 `materials.is_single_source = 1`，直接判定为唯一供应商
2. 否则按 `material_suppliers` 有效供应商数量判断
3. 若供应商数 = 1，则视为唯一供应商

说明：

- 第一阶段不做“有效期”判断，默认表内供应商都有效
- 后续可增加 `is_active` 做供应商启停

## 5.2 有无替代料判定

从 `material_substitutions` 判断：

- 存在 `is_active = 1` 的替代关系，则视为 `hasSubstitution = true`
- 否则为 `false`

如果后续需要更严谨，可继续结合：

- BOM/SOP 是否允许替代
- 工单是否允许替代

## 5.3 有效交期判定

优先级：

1. 默认供应商 `lead_time_days`
2. 物料主档 `lead_time_days`
3. 若都没有，按 `0`

记为：

- `effectiveLeadTimeDays`

## 5.4 覆盖天数判定

第一阶段建议支持两种模式：

### 模式 A：手工目标保供天数

适用于暂时没有稳定消耗模型的阶段。

使用：

- `coverage_days_target`

仅比较：

- 当前库存是否足以支撑目标保供量 / 保供套数

### 模式 B：静态覆盖天数估算

如果后续补充了日均消耗，可计算：

- `coverageDays = currentStock / avgDailyConsumption`

第一阶段允许预留，不强制上线。

---

## 6. 风险评分规则

第一阶段建议用规则加权分，不做机器学习。

建议默认分值如下：

- 唯一供应商：`+40`
- 默认供应商交期 >= 7 天：`+20`
- 默认供应商交期 >= 14 天：`+30`
- 默认供应商交期 >= 30 天：`+40`
- 无替代料：`+25`
- 关键件：`+25`
- 当前低于安全库存：`+20`
- 当前低于补货点：`+15`
- 当前无法满足目标保供套数：`+30`
- 来源平台为 `marketplace / retail / other`：`+10`

建议风险等级：

- `0 - 29`：普通
- `30 - 59`：中风险
- `60 - 89`：高风险
- `>= 90`：极高风险

返回字段建议：

- `riskScore`
- `riskLevel`
- `riskReasons[]`

例如：

```json
{
  "riskScore": 115,
  "riskLevel": "critical",
  "riskReasons": [
    "唯一供应商",
    "交期 10 天",
    "无替代料",
    "关键件",
    "当前低于目标保供套数"
  ]
}
```

## 7. 预警触发逻辑

## 7.1 普通低库存预警

继续沿用当前规则：

- 优先 `safety_stock`
- 为空时回退 `min_stock`

## 7.2 保供套数预警

继续沿用当前规则：

- 有 BOM 的输出物料
- 当前可生产套数 `< target_coverage_qty`

## 7.3 高风险供应预警

新增规则：

满足以下任一条件时进入高风险列表：

1. `riskScore >= 高风险阈值`
2. `is_single_source = 1` 且 `effectiveLeadTimeDays >= 7`
3. `is_key_part = 1` 且 `hasSubstitution = false`
4. 当前库存可支撑天数 `< effectiveLeadTimeDays + safetyBufferDays`
5. 当前已触发保供套数预警，且关键瓶颈物料属于唯一供应商或长交期物料

默认缓冲：

- `safetyBufferDays = 3`

可后续配置。

## 8. 页面与交互落点

## 8.1 仪表盘

新增卡片：

- `高风险供应预警`

展示：

- 数量
- 极高风险数
- 说明：`优先处理唯一供应商、长交期、无替代料关键件`

点击后进入：

- `高风险供应治理页`

## 8.2 报表 / 治理

新增治理页或新增治理 Tab：

- `高风险供应治理`

支持筛选：

- 风险等级
- 是否唯一供应商
- 是否关键件
- 是否有替代料
- 来源平台
- 交期区间

表格字段建议：

- 物料编码
- 物料名称
- 默认供应商
- 来源平台
- 交期
- 是否唯一供应商
- 是否关键件
- 是否有替代料
- 当前库存
- 安全库存
- 保供套数缺口
- 风险等级
- 风险原因

行内操作建议：

- 查看物料
- 查看 BOM / 工单引用
- 查看替代料
- 查看动作流水

## 8.3 物料详情页

新增或强化显示：

- 供应风险等级
- 是否唯一供应商
- 默认交期
- 是否有替代料
- 最近采购参考
- 默认供应商来源平台

## 8.4 工单详情页

如果工单需求中存在高风险物料：

- 顶部显示风险提醒
- 高亮对应物料
- 提示原因：唯一供应商 / 长交期 / 无替代料 / 关键件

## 9. API 设计建议

### 9.1 统计接口

新增：

- `GET /api/statistics/supply-risk-summary`
- `GET /api/statistics/supply-risk-items`

返回：

- 汇总数
- 风险等级统计
- 物料清单
- 风险原因

### 9.2 物料详情接口增强

在 `GET /api/materials/:id` 增加：

- `supplyRisk`
  - `riskScore`
  - `riskLevel`
  - `reasons`
  - `supplierCount`
  - `effectiveLeadTimeDays`
  - `hasSubstitution`

### 9.3 可生产套数接口联动

在现有套数预警逻辑里，增加瓶颈料附加字段：

- `isSingleSource`
- `effectiveLeadTimeDays`
- `riskLevel`

这样套数预警里能直接看出：

- 到底缺了什么
- 缺的料是不是高风险供应料

## 10. 开发实施顺序

### 第一阶段：先把规则算出来

1. 增加物料字段
2. 补供应商风险评分逻辑
3. 新增高风险供应统计接口
4. 仪表盘新增卡片

### 第二阶段：把治理入口做出来

1. 高风险供应治理列表
2. 筛选与导出
3. 一键查看物料 / 流水 / 替代料

### 第三阶段：和生产闭环联动

1. 工单详情高亮高风险物料
2. 套数预警结合风险料显示
3. BOM / 工单缺料视图显示风险原因

---

## 11. 验收标准

满足以下条件可判定第一版上线：

1. 能正确识别唯一供应商物料
2. 能正确识别交期 >= 7 天的料
3. 能正确识别关键件和无替代料
4. 仪表盘可显示高风险供应预警数量
5. 可进入治理列表查看风险原因
6. 可在物料详情看到完整供应风险信息
7. 套数预警中的瓶颈料可显示供应风险标签

---

## 12. 当前实现建议

基于当前系统现状，建议优先做以下最小可用版本：

### 必做

- 新增：
  - `is_single_source`
  - `coverage_days_target`
  - `supply_risk_level`
  - `supply_risk_notes`
- 后端计算：
  - `supplierCount`
  - `effectiveLeadTimeDays`
  - `hasSubstitution`
  - `riskScore`
  - `riskLevel`
- 仪表盘新增：
  - `高风险供应预警`
- 报表新增：
  - `高风险供应治理`

### 暂不强制

- 日均消耗自动计算
- 动态覆盖天数预测
- 采购建议自动生成
- 供应商有效期管理

---

## 13. 一句话决策原则

对于“唯一供应商 + 长交期 + 无替代料 + 关键件”的物料：

- 不应等到普通低库存触发才报警
- 应提前纳入高风险供应预警
- 并在保供套数、工单缺料、采购治理中优先显示
