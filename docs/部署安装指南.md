# OvO System 部署安装指南

> 本文档面向 AI 代理（Codex/OpenClaw）阅读并执行。
> 目标：在一台 24 小时运行的 Windows 电脑上完成 OvO System 物料管理系统的完整部署，包括系统本体和 MCP Server。

---

## 前置条件

- 操作系统：Windows 10/11
- 网络：能访问互联网（安装依赖时需要）
- 部署包文件：`OvO System-v1.0.zip`（已提供）

---

## 第一步：安装 Node.js

检查是否已安装 Node.js：

```powershell
node -v
npm -v
```

如果上述命令报错"不是内部或外部命令"，需要安装 Node.js：

1. 下载 Node.js LTS 版本：https://nodejs.org/zh-cn/download
2. 推荐版本：20.x LTS 或 22.x LTS
3. 安装时**必须勾选** "Add to PATH"
4. 安装完成后重新打开终端，再次执行 `node -v` 确认

要求：Node.js >= 18.0.0

---

## 第二步：解压部署包

将 `OvO System-v1.0.zip` 解压到目标目录，推荐路径：

```
C:\OvO System
```

解压后的目录结构应为：

```
C:\OvO System\
├── server\          # 后端服务
├── public\          # 前端页面
├── mcp-server\      # AI 接入层（MCP Server）
├── deploy\          # 部署脚本
├── docs\            # 文档
├── data\            # 数据库（首次运行自动创建）
├── package.json
└── .env.example
```

---

## 第三步：安装项目依赖

在终端中执行：

```powershell
cd C:\OvO System
npm install --production
```

如果在国内下载慢，先设置镜像：

```powershell
npm config set registry https://registry.npmmirror.com
npm install --production
```

安装 MCP Server 依赖：

```powershell
cd C:\OvO System\mcp-server
npm install --production
cd C:\OvO System
```

安装 Windows 服务管理工具（用于开机自启）：

```powershell
npm install --save-dev node-windows
```

---

## 第四步：配置环境变量

复制模板文件创建 `.env`：

```powershell
copy .env.example .env
```

编辑 `.env` 文件，内容如下：

```env
PORT=3000
SESSION_SECRET=修改为一段随机字符串至少32位
NODE_ENV=production
```

SESSION_SECRET 生成方法（PowerShell）：

```powershell
-join ((65..90) + (97..122) + (48..57) | Get-Random -Count 40 | ForEach-Object {[char]$_})
```

将输出的随机字符串粘贴到 SESSION_SECRET= 后面。

## 第六步：开放防火墙端口

以**管理员权限**打开终端，执行：

```powershell
netsh advfirewall firewall add rule name="OvO System (TCP 3000)" dir=in action=allow protocol=tcp localport=3000
```

或者直接右键**以管理员身份运行** `deploy\open-firewall.bat`。

这一步让局域网内的其他电脑可以通过浏览器访问本系统。

在本机浏览器打开 http://localhost:3000 ，应看到登录页面。

用以下账号登录：
- 用户名：`admin`
- 密码：`admin123`

查看本机 IP：

```powershell
ipconfig | findstr "IPv4"
```

在局域网内另一台电脑的浏览器中访问 `http://本机IP:3000`，应同样看到登录页面。

## 第八步：注册为 Windows 开机自启服务

```powershell
cd C:\OvO System
node deploy\register-service.js
```

注册后系统会以 Windows 服务形式在后台运行，开机自动启动，无需手动打开终端。

管理服务：

```powershell
# 查看服务状态
sc query OvO System

# 停止服务
net stop OvO System

# 启动服务
net start OvO System

# 卸载服务（如需要）
node deploy\unregister-service.js
```

---

## 第九步：配置 MCP Server（AI 接入）

MCP Server 让 OpenClaw/Claude 能通过自然语言操作物料系统。

在 OpenClaw 的 MCP 配置文件中添加以下内容。

**配置文件位置**（根据使用的 AI 工具选择）：

- Claude Desktop: `%APPDATA%\Claude\claude_desktop_config.json`
- OpenClaw: 按 OpenClaw 文档中的 MCP 配置路径

**配置内容**：

```json
{
  "mcpServers": {
    "OvO System": {
      "command": "node",
      "args": ["C:\\OvO System\\mcp-server\\index.js"],
      "env": {
        "IMS_API_URL": "http://localhost:3000",
        "IMS_USERNAME": "ai-operator",
        "IMS_PASSWORD": "ai123456"
      }
    }
  }
}
```

> 注意：如果系统安装在其他路径，需将 `C:\\OvO System` 替换为实际路径。路径中的反斜杠必须双写（`\\`）。

配置完成后**重启 OpenClaw/Claude Desktop**，在工具列表中应看到 OvO System 的 23 个工具。

### 验证 MCP Server

在 OpenClaw 中输入以下指令测试：

```
查看当前物料系统的仪表盘数据
```

AI 应自动调用 `get_dashboard` 工具并返回物料总数、仓库数量等信息。

## 第十步：导入管家婆历史数据（可选）

如果需要从管家婆迁移历史数据：

1. 从管家婆导出"库存状况表"和"生产模板明细"两个 Excel 文件
2. 将文件放到 `C:\OvO System\data\` 目录下
3. 执行迁移脚本：

```powershell
cd C:\OvO System
node deploy\import-from-gjp.js --inventory "data\库存表.xlsx" --bom "data\生产模板.xlsx"
```

或者双击运行 `deploy\import-gjp.bat`（会提示输入文件路径）。

---

## 第十一步：设置自动备份（推荐）

以管理员权限运行：

```powershell
deploy\setup-auto-backup.bat
```

这会创建 Windows 计划任务，每天自动备份数据库到 `backups\` 目录。

手动备份：

```powershell
deploy\backup.bat
```

从备份恢复：

```powershell
deploy\restore-db.bat
```

## 常见问题

### npm install 报错网络超时

```powershell
npm config set registry https://registry.npmmirror.com
npm install --production
```

### 端口 3000 被占用

编辑 `.env` 文件，将 `PORT=3000` 改为其他端口（如 `PORT=8080`），同时更新防火墙规则和 MCP 配置中的端口号。

### 局域网其他电脑无法访问

1. 检查防火墙规则是否添加成功
2. 确认两台电脑在同一局域网/子网
3. 确认没有代理软件拦截（Clash/v2ray 等需设置局域网直连）

### 服务注册失败

如果 `register-service.js` 报错，可以用 `deploy\start.bat` 手动启动，效果相同，只是不会开机自启。

### 数据库损坏

从最近的备份恢复：

```powershell
deploy\restore-db.bat
```
