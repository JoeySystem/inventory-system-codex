# 产品化命名优化清单

目标：把系统中偏研发语气、内部缩写和阶段性表达，统一收敛成更接近仓库员、生产主管、业务人员日常理解的产品语言。

## 本轮已落地

- 一级导航：
  - `收` -> `入库`
  - `发` -> `出库`
  - `转` -> `调拨`
  - `盘` -> `盘点`
- 页面标题：
  - `收` 工作台 -> `入库`
  - `发` 工作台 -> `出库`
  - `转` 工作台 -> `调拨`
  - `盘` 工作台 -> `盘点`
  - `正式报表` -> `业务报表`
  - `一致性治理` -> `数据治理`
  - `SOP工艺` -> `SOP管理`
  - `BOM结构` -> `BOM管理`
- 搜索与快捷入口：
  - `进入收 / 发 / 转 / 盘` -> `进入入库 / 出库 / 调拨 / 盘点`
  - `收 / 发 趋势` -> `入库 / 出库趋势`
- 用户手册：
  - 导航说明与操作路径已同步改成新叫法

## 命名原则

- 优先使用用户动作语言：
  - `入库 / 出库 / 调拨 / 盘点`
- 查询页优先使用单据语言：
  - `采购收货单`
  - `库存调整单`
  - `生产领料单`
- 尽量避免研发过程词：
  - `阶段`
  - `标准入口`
  - `工作台`
  - `执行页`
- 尽量避免过短缩写：
  - `收 / 发 / 转 / 盘`

## 后续可继续优化

- `仓库资料与执行` 可进一步改成：
  - `仓库旧入口`
  - 或 `仓库过渡入口`
- `动作流水` 可视业务习惯评估是否改成：
  - `库存流水`
- `数据治理` 可按角色区分：
  - 管理层看 `数据治理`
  - 仓库员看 `异常治理`
- 部分卡片中的 `独立执行页 / 独立查询页` 可继续收敛为：
  - `录单入口`
  - `查单入口`

## 本轮不调整的内容

- 路由路径不改
- API 名称不改
- 数据库字段不改
- 单据状态命名不改

这样可以在不影响部署、接口和已有链接的前提下，先完成用户界面的产品化表达收口。
