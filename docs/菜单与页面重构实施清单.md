# 菜单与页面重构实施清单

## 1. 文档目的

本文档用于将“收发转盘前台 + 进销存后台”新架构落成可执行的菜单与页面改造清单。

本文档不再讨论方案合理性，而是直接回答：

- 先改哪些菜单
- 先改哪些页面
- 每一阶段产出什么
- 哪些页面保留，哪些页面拆分
- 哪些改造可以只改入口，哪些必须改交互和数据结构
- 每一阶段如何验收

本文档面向：

- 产品经理
- 前端开发
- 后端开发
- 测试
- 业务验收人员

## 2. 实施目标

本次菜单与页面重构的目标是：

- 让仓库员和车间主管按 `收 / 发 / 转 / 盘` 理解和操作系统
- 保留 `物料 / BOM / SOP / 生产` 这些专业能力
- 不在第一阶段破坏原有数据结构与业务闭环
- 用最小改动先完成信息架构升级
- 为第二阶段单据模型与库存流水改造预留清晰入口

## 3. 改造范围

本清单覆盖：

- 左侧导航菜单
- 一级页面入口
- 二级菜单结构
- 页面标题与文案
- 页面内动作入口
- 旧页面到新入口的映射

本清单暂不覆盖：

- 全量数据库重构
- 全量单据模型重写
- 财务核算模块
- 审批流引擎

## 4. 改造总原则

### 4.1 先改入口，再改内核

先调整：

- 菜单
- 页面归类
- 用户看到的操作路径

后调整：

- 单据模型
- 库存流水
- 报表映射

### 4.2 先保留旧能力，再做结构归位

原则上：

- 不删除当前已有业务页面
- 先把页面放到更合理的位置
- 再逐步合并或拆分

### 4.3 页面重构分三类

三类页面的处理方式不同：

#### A. 直接保留型

只改菜单位置和标题，不改核心逻辑。

适用：

- 物料管理
- BOM管理
- SOP管理
- 用户管理

#### B. 归类调整型

页面能力保留，但入口归入新模块。

适用：

- 发货管理
- 仓库管理中的入出调盘动作页

#### C. 中枢增强型

页面保留并增强，需要后续驱动其他动作单据。

适用：

- 生产管理

## 7. 分阶段实施清单

## 第一阶段：导航与标题重构

### 7.1 目标

- 让用户先看到新的业务结构
- 不改变现有页面主体逻辑

### 7.2 页面动作

需要修改：

- 侧边栏导航配置
- 路由标题
- 页面头部标题
- 面包屑
- 用户手册导航说明

### 7.3 本阶段具体任务

1. 将 `物料管理` 菜单标题改为 `物料`
2. 新增一级导航：
   - `收`
   - `发`
   - `转`
   - `盘`
   - `生产`
   - `报表`
   - `系统`
3. 将 `SOP管理 / BOM管理 / 生产管理` 归入 `生产`
4. 将 `数据统计 / 报表中心` 归入 `报表`
5. 将 `用户管理` 归入 `系统`
6. 保留 `仓库管理 / 发货管理` 原页面，但从一级入口迁出

### 7.4 产出

- 新导航结构
- 页面标题统一
- 用户感知到新的业务分区

### 7.5 验收标准

- 左侧菜单结构与目标架构一致
- 现有页面功能不受影响
- 用户能从新入口进入旧页面

## 第二阶段：仓库管理拆分

### 8.1 目标

- 把当前“仓库管理”从混合入口拆成：
  - 仓库资料
  - 收发转盘执行入口

### 8.2 要拆出的能力

#### 保留到系统层

- 仓库主数据维护
- 仓库启停用
- 仓库基础配置

#### 拆到动作层

- 入库相关动作 -> 收
- 出库相关动作 -> 发
- 调拨相关动作 -> 转
- 盘点相关动作 -> 盘

### 8.3 页面改造任务

1. 新建 `收` 模块主页
2. 新建 `发` 模块主页
3. 新建 `转` 模块主页
4. 新建 `盘` 模块主页
5. 将当前仓库页中的操作按钮重新链接到对应动作模块
6. 将仓库主数据管理迁到 `系统 > 仓库资料`

### 8.4 短期实现建议

短期可采用：

- 新建四个工作台页面
- 每个工作台先放当前功能入口卡片
- 不强制立即重写原始功能页

### 8.5 验收标准

- 仓库员不再依赖“仓库管理”完成日常作业
- “仓库管理”只剩基础资料职责

## 第三阶段：发货管理归入发模块

### 9.1 目标

- 将“发货管理”统一纳入 `发`

### 9.2 页面改造任务

1. 一级菜单删除 `发货管理`
2. 在 `发` 下新增二级入口：
   - 销售发货
3. 原发货列表页保留
4. 原发货详情页保留
5. 页面标题从“发货管理”调整为：
   - 销售发货
   - 发货记录

### 9.3 后续预留

后续可在 `发` 下继续增加：

- 生产领料
- 样机借出
- 报废出库

### 9.4 验收标准

- 发货能力完整保留
- 仓库员能在 `发` 中找到发货入口

## 第四阶段：生产域归一

### 10.1 目标

- 将 `SOP / BOM / 生产管理` 收敛到统一的生产域

### 10.2 页面结构

建议在 `生产` 下提供三个二级入口：

- 生产工单
- SOP
- BOM

### 10.3 页面改造任务

1. 将 `生产管理` 改名为 `生产工单`
2. 将 `SOP管理` 改名为 `SOP`
3. 将 `BOM管理` 改名为 `BOM`
4. 新建 `生产` 域首页，展示：
   - 待领料工单
   - 生产中工单
   - 待完工入库工单
   - 缺料工单
5. 在 `生产工单` 页补充动作入口说明：
   - 开始生产 -> 触发发料
   - 完工 -> 触发收货

### 10.4 本阶段注意事项

- 不要将 SOP 或 BOM 强行改成收发转盘页面
- 不要删除现有工单能力
- 先改入口和标题，再做流程联动

### 10.5 验收标准

- 用户能明确区分：
  - SOP 是工艺
  - BOM 是结构
  - 工单是执行

## 第五阶段：报表域归一

### 11.1 目标

- 将 `数据统计` 与 `报表中心` 统一到 `报表` 域

### 11.2 页面改造任务

1. 报表域首页增加卡片：
   - 动作报表
   - 进销存报表
   - 库存价值
   - 生产分析
2. 原 `数据统计` 保留为分析看板
3. 原 `报表中心` 保留为正式导出与核算报表
4. 后续补充：
   - 收发转盘动作日报
   - 盘点差异报表
   - 生产领料与完工对比报表

### 11.3 验收标准

- 经营报表与动作报表分层清楚
- 原报表功能继续可用

## 第六阶段：页面内交互重构

### 12.1 目标

- 在不彻底重写页面的前提下，让页面内部也体现新架构语言

### 12.2 页面内改造重点

#### 仪表盘

- 增加“今日收 / 发 / 转 / 盘”摘要
- 保留低库存、库存价值、报表摘要

#### 生产工单页

- 增加动作提示
- 增加工单状态与动作映射文案

#### 发货页

- 用“发”视角重写页面说明

#### 仓库相关页

- 明确区分：
  - 仓库资料
  - 仓库作业

### 12.3 验收标准

- 页面不只是换菜单，还体现新业务语言

## 10. 页面级验收清单

### 10.1 菜单验收

- 一级菜单符合目标结构
- 所有现有核心页面仍可到达
- 新入口不出现死链

### 10.2 页面归类验收

- 发货页已并入发
- SOP / BOM / 生产已归入生产
- 数据统计 / 报表中心已归入报表
- 用户管理已归入系统

### 10.3 业务理解验收

- 仓库员能根据动作语言找到入口
- 车间主管能根据生产语言找到入口
- 管理层能根据报表语言找到入口

## 11. 推荐实施顺序

建议按以下顺序推进：

1. 导航重构
2. 生产域聚合
3. 发货归发
4. 仓库页拆分
5. 收发转盘工作台
6. 报表域整理
7. 仪表盘动作化
8. 页面文案统一

## 12. 交付物清单

本阶段实施完成后，应形成以下交付物：

- 新菜单信息架构
- 页面映射表
- 页面标题与文案清单
- 路由调整清单
- 角色入口说明
- 页面验收清单

## 13. 结论

菜单与页面重构的核心不是“删掉旧模块”，而是：

- 让一线人员先看到动作语言
- 让专业能力继续保留在正确的位置
- 让系统逐步从“功能堆叠”变成“层次清楚的业务平台”

本实施清单建议先从导航与入口改起，再逐步收口到页面内动作、工单联动和报表统一。
